title 8. Two-Tier Gemini → RC Physics Routing

participant Caller
participant Router
participant Gemini
participant Circuit Breaker
participant RC Physics

Caller->Router: simulate_step_with_hvac()\nor generate_hvac_schedule()

Router->Circuit Breaker: Quota blackout active?
Circuit Breaker-->Router: YES → skip Gemini for 60 min

alt Blackout active
    Router->RC Physics: _physics_step() or _physics_schedule()
    RC Physics-->Router: result {engine:"physics"}
    Router-->Caller: physics result
end

Router->Gemini: POST prompt\n(step or schedule)\ntemperature=0.1, JSON mode

alt 429 / RESOURCE_EXHAUSTED
    Gemini-->Router: quota error
    Router->Circuit Breaker: Trip → block 60 min
    Router->RC Physics: fallback
    RC Physics-->Router: result
    Router-->Caller: physics result
end

alt JSON parse fails
    Gemini-->Router: malformed response
    Router->Router: strip fences, retry on substring
    Router->RC Physics: fallback if both attempts fail
    RC Physics-->Router: result
    Router-->Caller: physics result
end

Gemini-->Router: valid JSON

Router->Router: Step only — template-copy check:\nif mode=off AND ΔT≈0\nAND gap > deadband → reject

alt Template copy detected
    Router->RC Physics: fallback
    RC Physics-->Router: result
    Router-->Caller: physics result
end

note over Router: Schedule only — energy floor check:\nif ai_kwh < physics_floor×0.20 → reject\n(house can't heat itself on near-zero kWh)

Router-->Caller: Gemini result {engine:"genai"}
