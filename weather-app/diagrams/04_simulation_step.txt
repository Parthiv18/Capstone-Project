note over Frontend: Polls every 5 minutes

FastAPI->SQLite: get_user_state(username)\nSELECT users JOIN user_thermostat
SQLite-->FastAPI: {house, weather, simulated_temp,\nhvac_sim, target_setpoint, last_updated}

FastAPI->FastAPI: Resolve setpoint (priority chain):\n1. Manual override (if active 60 min)\n2. AI-managed (if enabled, refresh every 30 min)\n3. Saved DB setpoint\n4. personal_comfort from house\n5. Default 22°C

FastAPI->FastAPI: Select weather row closest to now\nΔt = min(elapsed since last_updated, 300s)

FastAPI->Gemini: POST step prompt\n{house geometry, T_in, T_target, T_out,\nREQUIRED MODE (pre-computed), TOU rate,\nRC physics context, deadband analysis}

alt Gemini valid response
    Gemini-->FastAPI: {new_temp_c, hvac_mode,\nhvac_power_kw, cost_this_step, reason}
    FastAPI->FastAPI: Reject if mode=off AND ΔT≈0\nAND gap > deadband\n(template-copy detection)
end

    RC Physics->RC Physics: Q_cond + Q_wind + Q_solar + Q_HVAC\nDeadband → mode\nT_new = T_in + Q_total×Δt / C_home
    RC Physics-->FastAPI: {new_temp_c, hvac_mode,\nq_conductive_w, q_wind_w,\nq_solar_w, engine:"physics"}
end
